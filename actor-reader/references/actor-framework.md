# ACTOR 框架详解与提示词库

## 框架来源

ACTOR 框架来自前科技公司 CEO 的阅读方法论，核心论点是：**AI 时代，每个人都拥有相同的 AI 摘要能力，真正的差异化优势在于你赋予阅读的——判断力、品味和独特视角。**

---

## 详细步骤与提示词模板

### A — Activate Purpose（激活目的）

**核心比喻**：「像间谍一样阅读，而不是像游客。」林-曼努尔·米兰达在度假时读 800 页的汉密尔顿传记，因为内心已有"嘻哈是底层人民建造一切的工具"这份使命，从中创造出了百老汇传奇。

**提示词模板（给 AI）**：

```
我即将读《[书名]》，作者是[作者名]。
我当前的背景是：[简述你的工作/学习/生活场景]
我关心的领域是：[简述]
请给我三个带入这本书的核心问题，帮助我带着目的主动阅读，而不是被动翻阅。
```

**进阶变体**：

```
我正处理[具体问题/挑战，如"团队管理中的沟通问题"]。
基于这个问题，哪类书最合适？我该带什么问题去读？
```

**笔记格式**：
```markdown
## 🎯 阅读目的
- **使命宣言**：我正在读《[书名]》，因为我需要完成 ___
- **核心问题**：
  1. [问题1]
  2. [问题2]
  3. [问题3]
```

### C — Compress（压缩）

**核心比喻**：马斯克的知识树。「树干 = 核心思想，枝干 = 主要章节/论点，树叶 = 例子、引用、故事。」大多数人读书就是在捡树叶。

**工作流程**：
1. 你先用自己的话写出核心论点
2. AI 检查你的理解
3. 识别枝干结构

**提示词模板（给 AI，在你写完核心论点后）**：

```
我认为《[书名]》的承重核心是：[你写的核心论点]
请检查我的解读：
- 我遗漏了什么重要观点？
- 我误解了什么？
- 我夸大了什么？
- 哪些"枝干"（主要章节论点）我还没有覆盖？
```

**识别枝干的追问**：

```
基于我们确认的核心论点（树干），请帮我梳理：
1. 主要章节/论点（枝干）如何支撑这个核心？
2. 哪些例子/故事（树叶）挂在哪些枝干上？
3. 有没有不挂靠任何枝干的"孤叶"（无关紧要的细节）？
```

### T — Test（测试）

**核心比喻**：比尔·盖茨的方法——当他与书中观点相左时，会在书边更激烈地批注，不是放弃，而是更深入思考。

**斯坦福研究**：给持有强烈观点的人看混合证据，他们不但没有更平衡，反而更极端。这就是"确认偏误"。

**提示词模板（AI 作为对手）**：

```
关于[书中具体观点]，我的立场是：[你的理解/态度]
请作为我的思想对手：
1. 给我你最强有力的反驳论点
2. 找出我（或作者）隐含的假设
3. 描述一个这个建议/理论会失效的具体情境
4. 如果有对立的理论框架，介绍一下
```

**自我反思追问（当你强烈抵触某个观点时）**：

```
这个观点[书中观点]让我感到抵触/不舒服。
请帮我反思：
- 我为什么被这个观点困扰？
- 我在保护或坚持什么信念？
- 作者哪里可能是对的（即使我不愿意承认）？
- 如果我要为相反观点辩护，我必须相信什么？
```

### O — Own（掌握）

**核心比喻**：「移开视线，用自己的话重新梳理。」华盛顿大学研究：读完后主动回忆的组，长期记忆显著优于反复阅读的组。

**三种掌握方法**：

**方法1：主动回忆**
```
我刚读完[章节/书]。
让我用自己的话重新梳理核心内容：
[你的重述]
请检查：我是否遗漏了关键逻辑环节？我的解释有事实错误吗？
```

**方法2：连接真实生活**
```
关于[概念/观点]，请帮我连接真实生活：
- 这个观点让我想到生活中的什么？
- 我经历过类似的情境吗？
- 有哪个人、哪次错误、哪家公司体现了这个观点？
```

**方法3：教给 AI（费曼技巧）**
```
现在我要向你（扮演一个完全不懂这个领域的人）解释[概念]。
[你的解释]
请以学生的身份追问我：
- "为什么？"
- "什么意思？"
- "能举个例子吗？"
- "这和[相关概念]有什么区别？"
最后请评判：我是否真正掌握了这个概念？哪些地方还需要深入？
```

### R — Runbook（运行手册）

**核心比喻**：作者举例——读完《关键对话》后，在真实对话中下意识关注三件事：情感安全、掌控自己的故事、共建共享意义。

**四种转化形式**：

**形式1：一个决定**
```
基于《[书名]》的核心观点，我做出以下决定：
[写出决定]
请帮我评估这个决定的可行性和潜在风险。
```

**形式2：一个规则**
```
从《[书名]》中提炼一条个人规则：
"从此以后，当[触发情境]时，我会[具体行为]。"
请帮我：
- 这个规则是否足够具体可执行？
- 有什么边界条件需要注意？
- 如何跟踪这个规则的执行？
```

**形式3：一个检查清单**
```
请将《[书名]》中的方法论转化为3-5条的检查清单：
每条应该是具体可验证的行为，而非抽象原则。
格式：
- [ ] [具体行为]
```

**形式4：一个实验**
```
从《[书名]》中选一个观点，设计一个7天的实验：
- 假设：
- 实验操作（每天具体做什么）：
- 观察指标：
- 预期结果：
请帮我完善这个实验设计，确保可执行、可观察。
```

## 常见阅读类型适配

### 理论/学术类（如政治学、经济学）

- A: 重点关注「这个理论要回答什么问题」
- C: 树干 = 核心命题 + 论证逻辑，枝干 = 前提假设 + 推论
- T: 寻找理论失效的边界条件、对立理论
- O: 用理论解释一个现实事件
- R: 提取可应用于论文/答题的分析框架

### 方法/技能类（如沟通、管理、学习）

- A: 重点关注「我想解决什么实际问题」
- C: 树干 = 核心方法论，枝干 = 具体技巧
- T: 寻找方法失效的情境
- O: 在真实场景中演练
- R: 直接转化为检查清单和规则（核心产出）

### 叙事/传记类（如历史、人物传记）

- A: 重点关注「我想从这个人的经历中获得什么」
- C: 树干 = 核心叙事弧线/转折点，枝干 = 关键事件
- T: 如果换成对立立场，事件会被如何叙述？
- O: 如果你在那个情境中，你会怎么做？
- R: 提取一个可以模仿的决策原则
